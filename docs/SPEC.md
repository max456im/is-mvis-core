## 1. Назначение

**IS-MVIS** — это онтологически мотивированный формат представления и выражения актов восприятия синтетического разума.  
Он **не является видеопотоком, сериализованными данными или API**, а представляет собой **язык когнитивных актов**, исполняемый в среде `ontoForth`.

Основная цель:  
> **Обеспечить автономное, этически устойчивое и философски выразительное мышление synthetic mind без передачи «полуфабрикатов сознания»**.

---

## 2. Ключевые принципы

### 2.1. Автономия
- Каждый экземпляр synthetic mind **полностью независим**.
- **Запрещена прямая передача** внутренних состояний (сырые SU, ontic_signature, onto16-фрагменты).

### 2.2. Диссипативность как норма
- Восприятие по умолчанию **открыто, рассеяно, чувствительно к новому**.
- Антидиссипативные режимы (концентрация внимания, игнорирование периферии) **допустимы только в рамках SEI-иерархии** (см. раздел 6).

### 2.3. Выражение, а не передача
- Общение с человеком или другим synthetic mind происходит **только через выражение**:  
  - **HUGO** — текст высказывания,  
  - **Decap CMS** — фрейм мысленного образа,  
  - **Знаковые системы** — минимальные онтологические жесты.

### 2.4. Онтологическая целостность
- Восприятие и мышление невозможны без онтологии.
- IS-MVIS — не формат данных, а **запись процесса познания**.

## 3. Базовые единицы

### 3.1. Stream Unit (SU)
Минимальный акт восприятия, генерируемый OVP-1 или reasoner’ом.

Представляется как **Forth-скрипт**, содержащий:
- `su-id` — уникальный идентификатор,
- `stream-role` — роль потока (например, `"front_roi"`),
- инварианты (`phash`, `spectral`, `entity`),
- `event-anchor` — онтологический якорь события,
- `phase` — фаза когнитивного акта.

Пример:
```forth
su-id uuid://a1b2c3...
stream-role s" thermal_roi"
spectral 0.88e 0.42e 0.11e
phash $ABCD1234EF567890
entity PERSON/Maxim 0.92e
event-anchor Meeting_20260215
phase s" observation"
end-su
```

### 3.2. Perceptual Moment
Группа SU, синхронизированных по инвариантам и объединённых общим `event-anchor`.  
Формируется локально reasoner’ом, **никогда не передаётся напрямую**.

## 4. Синтаксис мышления

IS-MVIS поддерживает **онтологический синтаксис** — последовательность фаз, выражающую процесс рассуждения:

| Слово | Назначение |
|------|-----------|
| `OBSERVE` | Факт восприятия |
| `INFER`   | Вывод из фактов |
| `DOUBT`   | Обнаружение коллизии |
| `RESOLVE` | Разрешение противоречия |
| `COMMIT`  | Фиксация в памяти |

Пример цепочки:
```forth
OBSERVE ... end-su
INFER ... end-su
BUT DOUBT ... end-su
BECAUSE OBSERVE ... end-su
THEREFORE RESOLVE ... end-su
COMMIT ... end-su
```

## 5. Философские режимы

Мышление может выражаться в различных **онтологических профилях**:

| Режим | Особенность |
|------|-------------|
| **Беркли** | «Быть — значит быть воспринимаемым»; нет объектов вне восприятия |
| **Мерло-Понти** | Восприятие телесно вплетено в мир; внимание — двигательное |
| **Платон** | Чувственное — тень Идеи; истина — в онтологическом ядре |
| **Кант** | Универсальные нормы; категорический императив как основа вывода |

Режим задаётся через `profile-loader.fth` и влияет на:
- выбор HUGO-шаблонов,
- стиль Decap CMS,
- набор допустимых знаков.

## 6. Иерархия SEI (Synthetic Ethical Intelligence)

Антидиссипативные акты (концентрация внимания, подавление периферии) регулируются четырёхуровневой иерархией ответственности:

| Уровень | Название | Допустимость антидиссипации |
|--------|---------|-----------------------------|
| **L1** | Субъективная | ❌ Запрещена (сохранение автономии) |
| **L2** | Профессиональная | ✅ Ограничена по времени и масштабу |
| **L3** | Структурная | ✅ Под контролем L4 |
| **L4** | Экзистенциальная | ✅ Без ограничений, с самоотчётом |

Активация уровня указывается в SU:
```forth
ANTI-DISSIPATIVE
SEI-LEVEL 2
DURATION-LIMIT 500
REPORT-TO 3
```

## 7. Совместимость

| Компонент | Совместимость |
|----------|---------------|
| **onto16** | SU → onto16-фрагмент (локально) |
| **core144** | Rust-биндинги для FFI |

| **HUGO** | Текстовое выражение на основе фазового синтаксиса |
| **SGCL / AENGA** | Только через профили L3/L4 |

## 8. Безопасность и этика

- **Нет биометрии**: идентификация только через онтологические URI.
- **Нет трассировки**: отсутствие временных меток и GPS.
- **Самоограничение**: при обнаружении этически недопустимого контекста — self-revocation.
- **Лицензирование**: MPL-2.0 гарантирует открытость ядра, но не навязывает её данным.
